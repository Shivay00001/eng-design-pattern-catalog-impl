A **reference implementation of classic software design patterns**. This repository provides clean, pythonic examples of Creational, Structural, and Behavioral patterns to serve as a catalog for software engineers.

### Creational

- **Singleton**: Ensures a class has only one instance.
- **Factory Method**: Creates objects without specifying the exact class.

### Structural

- **Adapter**: Allows incompatible interfaces to work together.
- **Decorator**: Dynamically adds behavior to an object.

### Behavioral

- **Observer**: Notifies dependents of state changes.
- **Strategy**: Encapsulates interchangeable algorithms.
